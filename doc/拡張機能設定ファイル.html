Extension Configuration with properties.json

各拡張機能には、WebFOCUS®がユーザーインターフェースを描画する際に必要な情報を定義するproperties.jsonファイルが必要です。このファイルは人間が読める JSON 形式で、非常に分かりやすい構造になっています。

以下は、Simple Barデモ拡張機能の完全なproperties.jsonです。各ブロックの詳細は以下の通りです：

{
	// 一般的な拡張機能の設定オプションを定義
	"info": {
		"version": "1.0",  // 拡張機能のバージョン番号
		"implements_api_version": "1.0",  // 拡張機能が使用するWebFocus(R) APIのバージョン番号
		"author": "Information Builders",
		"copyright": "Information Builders Inc.",
		"url": "https://github.com/ibi/wf-extensions-chart/tree/master/simple_bar%20example",
		"icons": {
			"medium": "icons/medium.png"  // WFチャートピッカーで使用される拡張機能内の画像への参照
		}
	},
	
	// エンドユーザーが変更したい可能性のある拡張機能のプロパティを定義
	"properties": {
		"exampleProperty": 50
	},
	
	// 'properties'の各プロパティで取り得る値を定義
	"propertyAnnotations": {
		"exampleProperty": "number"
	},

	// WFの'Query'データバケットツリーに表示される利用可能なデータバケットを定義
	"dataBuckets":  {
	
		// 既存のWFデータバケットを再利用するかどうかを選択。すべてオプション
		"tooltip": false,
		"series_break": true,

		// 独自のカスタムデータバケットを定義。オプション
		"buckets": [
			{
				"id": "value",
				"type": "measure",
				"count": {"min": 1, "max": 5}
			},
			{
				"id": "labels",
				"type": "dimension",
				"count": {"min": 1, "max": 5}
			}
		]
	},

	// Define the set of labels used in the WF interface for buckets and chart type picker.
	"translations": {
		"en": {
			"name": "My Simple Bar Chart",
			"description": "This chart is just a simple bar chart, nothing to see here.",
			"icon_tooltip": "This extension does ...", 
			"value_name": "Value Bucket", 
			"value_tooltip": "Drop a measusre here", 
			"labels_name": "Label Bucket", 
			"labels_tooltip": "Drop a dimension here"
		},
		
		"fr": {
			"name": "Un Bar Chart tres simple",
			"description": "C'est un Bar Chart vraiment simple",
			"icon_tooltip": "This extension does ...", 
			"value_name": "Value Bucket", 
			"value_tooltip": "Drop a measusre here", 
			"labels_name": "Label Bucket", 
			"labels_tooltip": "Drop a dimension here"
		}
	}
}
info

infoブロックはいくつかの一般的な設定オプションを定義します。必須のプロパティはimplements_api_versionのみで、これは拡張機能が使用するWebFOCUS®拡張機能APIのバージョンと一致する必要があります。現時点での最新APIバージョンは"1.0"です。

properties と propertyAnnotations

propertiesとpropertyAnnotationsブロックは、エンドユーザーが変更したい可能性のある拡張機能のプロパティを定義するために使用されます。これらのプロパティは、FEXのGRAPH_JSブロックを通じてエンドユーザーに公開されます。例えば、com.ibi.simple_bar拡張機能のexamplePropertyプロパティを設定するには、ユーザーはFEXに以下を含める必要があります：

*GRAPH_JS_FINAL
	"extensions": {
		"com.ibi.simple_bar": {
			"exampleProperty": "HELLO!"
		}
	}
*END

propertyAnnotationsはpropertiesの内容を検証するために内部的に使用されます。propertiesのすべての項目はpropertyAnnotationsに出現する必要があります。propertiesのオブジェクトではないプロパティ（リーフプロパティ）の可能な型は、以下のいずれかとして注記する必要があります：

  • "str"
  • "bool"
  • "color"
  • "number"

より多くの注記オプションが利用可能ですが、このページの範囲を超えています。

"propertyAnnotations"ブロックの各プロパティの値は、WebFOCUS® Designerの拡張機能プロパティパネルに表示されるコントロールのタイプを決定します。これはプロパティ値として直接設定することも、その中に"typeAnnotation"プロパティを設定することもでき、コントロールの追加プロパティを指定できます。デフォルトでは、"str"の値はテキストボックス、"bool"はチェックボックス、"color"はカラーピッカー、"number"はスピナーを作成します。

"typeAnnotation"が"str"の場合、"stringValues"プロパティに値の配列を提供することで、それらの値を持つドロップダウンリストを作成できます。"typeAnnotation"が"number"の場合、デフォルトのスピナーステップは1で、最大値や最小値はありません。"numericRange"プロパティを使用して最小値と最大値を配列として設定し（例：[0,100]）、"numericStep"プロパティを使用して各値の間隔を設定します。これらのプロパティの両方が設定されている場合、デフォルトのコントロールはスライダーになります。代わりにスピナーを使用するように指定するには、プロパティ設定"uiType": "spinner"を使用します。例えば、"propertyAnnotations"ブロックの以下のプロパティセットは、0.05刻みのスピナーを使用して0から1の範囲でパディングを設定するコントロールを作成します。

"padding": {
	"typeAnnotation": "number",
	"numericRange": [0,1],
	"numericStep": 0.05,
	"uiType": "spinner"
},

他の2つの設定により、WebFOCUS® Designerに表示されるチャート拡張機能プロパティの順序と可視性を決定できます。プロパティ設定"private": trueを使用すると、WebFOCUS® Designerの拡張機能プロパティウィンドウからユーザーに見せたくない、または変更させたくないプロパティを非表示にできます。以下の構文例では、WebFOCUS® Designerの拡張機能プロパティウィンドウから"language"プロパティを非表示にします。

"propertyAnnotations": {
	"language": {
		"typeAnnotation": "str",
		"private": true
		},

チャート拡張機能のプロパティの表示順序も設定できます。"displayOrder"プロパティの値として整数を使用して、WebFOCUS® Designerの拡張機能プロパティウィンドウのプロパティリストでの表示位置を設定します。"displayOrder"の値が小さいほど、リストの上位に表示されます。例えば、"displayOrder"が0のプロパティは、"displayOrder"が1のプロパティの上に表示されます。以下の構文例は、properties.jsonファイルの4つのプロパティの注記を示しています。それぞれの"displayOrder"値により、"circleColor"プロパティは最も低い"displayOrder"を持つため拡張機能プロパティウィンドウで最初に表示され、続いて"circleThickness"プロパティと"waveCount"プロパティが表示されます。"circleFillGap"プロパティはプライベートに設定されているため非表示になっています。

"propertyAnnotations": {
	"circleThickness": {
		"typeAnnotation": "number",
		"displayOrder": 1	
			},
	"circleFillGap": {
		"typeAnnotation": "number",
		"displayOrder": 3,
		"private": true
			},
	"circleColor": {
		"typeAnnotation": "str",
		"displayOrder": 0
			},
	"waveCount": {
		"typeAnnotation": "number",
		"displayOrder": 2
			}
		}

WebFOCUS® Designerでリキッドゲージのプロパティを編集する際、プロパティは"displayOrder"と"private"設定に基づいて順序付けられ、表示または非表示になります。

dataBuckets

dataBucketsは、WFの'Query'バケットペインに表示されるデータバケットのセットを定義します。拡張機能が有用であるためには、少なくとも1つのバケットを定義する必要があります。

データバケットには2つのタイプがあります：built-inとcustomです。組み込みバケットは、WFの既存のデータバケットロジックを簡単に再利用する方法を提供します。現在、2つの組み込みバケットがあります：tooltipとseries_breakです。関連するdataBucketsプロパティをtrueに設定することで、これらのバケットのいずれかを使用できます。

bucketsの各エントリは1つのcustomデータバケットを定義します。各カスタムバケットには以下が必要です：

  • id：チャートのレンダリング関数が受け取るdataArrayMapとdataプロパティに正確に対応します。

  • type：このバケットが受け入れるデータフィールドのタイプを定義します。"measure"、"dimension"、"both"のいずれかです。

  • count：count.minとcount.maxは、このバケットが受け入れることができるフィールドの最小数と最大数を定義します。最小値が0の場合、このバケットはオプションです。

これらのバケットの使用と拡張機能に渡されるデータセットへの影響の詳細については、拡張機能データインターフェースページを参照してください。

translations

WFインターフェースに描画されるすべてのラベルには、様々な言語での翻訳を含める必要があります。translationsは、拡張機能がこれらのラベルを定義する場所です。translationオブジェクトには、拡張機能がサポートする各言語に対して1つのプロパティがあり、ISO-639の2文字のロケール文字列でキー付けされています。
