# タスク完了時のチェックリスト

## 拡張機能開発完了後の確認事項

### 1. コード品質チェック
- [ ] `renderConfig.renderComplete()` が適切に呼び出されている
- [ ] エラーハンドリングが実装されている
- [ ] データバリデーションが行われている
- [ ] コールバック関数が適切に実装されている
- [ ] 未使用の変数やコードがない

### 2. 設定ファイルの確認
- [ ] `properties.json` が正しく設定されている
- [ ] `info`、`properties`、`propertyAnnotations`、`dataBuckets`、`translations` が適切
- [ ] アイコンファイルが配置されている
- [ ] 必要な外部リソース（CSS/JS）が定義されている

### 3. テスト実行
- [ ] `test.html` でローカルテストが正常に動作する
- [ ] 複数のデータパターンでテストされている
- [ ] データがない場合の表示が適切
- [ ] エラー時の動作が適切

### 4. ドキュメント更新
- [ ] `README.md` が更新されている
- [ ] 使用方法が記述されている
- [ ] 依存関係が明記されている

### 5. デプロイ前チェック
- [ ] `html5chart_extensions.json` に拡張機能が追加されている
- [ ] 拡張機能IDが一意である
- [ ] ライセンス情報が適切に記述されている

## デプロイ手順

### 1. ローカルテスト
```bash
# テスト実行
npm run test
```

### 2. WebFOCUSへのデプロイ
```bash
# デプロイコマンド実行
npm run deploy [extension_id]
```

### 3. WebFOCUSでの確認
- [ ] Apache Tomcat再起動
- [ ] WebFOCUSの拡張グラフリストに表示される
- [ ] チャート作成で正常に動作する
- [ ] ブラウザキャッシュをクリアして確認

## 運用開始後のメンテナンス

### 1. 監視項目
- [ ] パフォーマンス（レンダリング速度）
- [ ] エラーログの確認
- [ ] ユーザーフィードバックの収集

### 2. アップデート手順
- [ ] バージョン情報の更新（`properties.json`）
- [ ] 変更履歴の記録
- [ ] 既存データとの互換性確認
- [ ] 段階的なロールアウト

## 品質基準

### パフォーマンス
- レンダリング時間 < 2秒（通常データサイズ）
- メモリリークの発生なし
- 大量データでの動作確認

### 互換性
- WebFOCUS各バージョンでの動作確認
- 主要ブラウザでの動作確認
- レスポンシブデザインの確認

### ユーザビリティ
- 直感的な操作性
- エラーメッセージの分かりやすさ
- ヘルプドキュメントの充実
